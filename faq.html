Frequently Asked Questions (FAQ)

This page collects common questions that someone new to SPEC might have, especially around installation, running the software, and using external resources like the camera or IMU.

1. What do I need installed before I can run SPEC?

SPEC is designed to install most of its own dependencies automatically, but you should have the following first:

  • A supported operating system (Linux / macOS / WSL on Windows).
  • Git so you can clone the repository.
  • A terminal that can run Bash scripts.

After that, you just run setup/set_up_system.sh, which installs the remaining software (Python packages, libraries, and tools used by SPEC).

2. I ran set_up_system.sh but I still get errors. What can I do?

The most common problems when running the setup script are:

  • Permission errors – try running chmod +x set_up_system.sh first.
  • Missing system package manager – make sure your OS has the package manager expected by the script (e.g., apt on Ubuntu).
  • Old Python version – verify that your Python version matches what SPEC expects.

Check the terminal output and the log file (if generated) for the exact message. If a specific dependency failed to install, you can usually fix it by installing that package manually and then re-running the script.

3. The camera / IMU is not detected. How do I fix this?

SPEC can run even without hardware, but to use live data you need the OS to see the devices first.

  • Make sure the camera or IMU is physically plugged in and powered.
  • On Linux, check that your user has permission to access the device (e.g., /dev/video0 or /dev/i2c-1).
  • Try unplugging and re-plugging the device, then restart SPEC.
  • If the IMU path is hard-coded, verify that the device name matches your system.

If the device is unavailable, SPEC will usually show a message like “IMU disabled” but other features (video processing, PIV) will still work.

4. PIV or video processing is very slow. Is that normal?

PIV and frame extraction can be compute-heavy, especially for long, high- resolution videos. You can speed things up by:

  • Using a shorter test clip while experimenting.
  • Reducing the resolution of the source video.
  • Lowering the frame rate during extraction (for example 10 fps instead of 60 fps).
  • Adjusting PIV parameters (larger interrogation windows or fewer passes).

Once you’re happy with the settings, you can run the full-resolution dataset.

5. Are there any common “gotchas” when using SPEC?

  • Wrong working directory: many scripts assume you are in the project root when running commands. If you get “file not found” errors, check your current directory.
  • Hard-coded paths: if you move videos or output folders, update the config or command-line arguments accordingly.
  • Case-sensitive filenames: on Linux, Data.mp4 and data.mp4 are different files, so make sure the names match exactly.

6. Are there any issues with external resources or services?

SPEC primarily relies on open-source libraries installed locally (e.g., Python packages for image processing and numerical analysis). Potential issues include:

  • Library version mismatches when running on a different machine than the one used for development.
  • OS-specific dependencies (for example, some camera libraries behave differently on Windows vs. Linux).

If you encounter a version-specific bug, note the OS and versions of Python and the failing library so it can be reproduced and fixed.
